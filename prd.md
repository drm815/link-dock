📑 [PRD] 링크독 (LinkDock)
"심플함이 제일 중요하다" — 개인화 맞춤형 링크 대시보드

1. 프로젝트 개요
* 프로젝트 이름: 링크독 (LinkDock)
* 한 줄 설명: 복잡한 북마크 대신, 업무와 일상을 완벽히 분리해 자주 가는 사이트를 한눈에 관리하는 미니멀 포털.
* 유형: 반응형 웹 대시보드 (Personal Portal)
* 난이도: 입문~초급 (React 입문 및 로컬 데이터 관리 학습에 최적)

2. 사용자 시나리오
* 누가: 공과 사를 확실히 구분하고 싶은 사용자, 브라우저 북마크바가 지저분해 정리가 필요한 사람.
* 언제: 출근 후 업무를 시작할 때(Work), 퇴근 후 휴식을 취할 때(Life), 혹은 모바일로 이동 중에 빠른 접속이 필요할 때.
* 왜: * 심플함: 군더더기 없는 UI로 필요한 정보만 노출.
    * 분리: 탭 하나로 업무 환경과 개인 환경을 즉시 전환.
    * 편의성: 모바일에서도 오클릭 없는 큼직한 버튼 제공.

3. 핵심 기능 목록
✅ 필수 기능 (Must-Have)
* 모드 전환(Work/Life): 상단 탭을 통해 업무용과 개인용 링크 목록을 필터링하여 노출.
* 링크 카드 시스템: 사이트명과 URL을 담은 큼직한 버튼형 카드 UI.
* 로컬 저장소(LocalStorage): 로그인 없이도 사용자가 등록한 링크가 브라우저에 영구 보존.
* 반응형 디자인: PC(그리드)와 모바일(단일 열) 환경에 맞춘 자동 레이아웃 최적화.
* 링크 추가/삭제: floating 버튼을 통해 간편하게 새로운 링크를 등록하고 관리.
✨ 선택 기능 (Nice-to-Have)
* 실시간 검색: 링크가 많아질 경우 제목이나 키워드로 즉시 필터링.
* 파비콘 자동 연동: URL 입력 시 해당 사이트의 아이콘을 자동으로 불러와 시각화.
* 데이터 백업: 현재 저장된 링크 리스트를 JSON 파일로 내보내거나 가져오기.

4. 기술 스택
* Framework: React (Vite) - 빠른 개발 및 배포 환경.
* UI Library: shadcn/ui - 깔끔하고 현대적인 컴포넌트 활용.
* Styling: Tailwind CSS - 강력한 반응형 레이아웃 구현.
* State Management: React Context API (Work/Life 모드 관리).
* Storage: window.localStorage (데이터 보존).
* Deployment: Vercel (언제 어디서든 접속 가능한 호스팅).

5. 화면 구성
1. Header: 좌측 상단 'LinkDock' 로고, 중앙/우측 'Work'와 'Life' 전환 탭.
2. Search Bar: 상단 탭 하단에 배치된 심플한 텍스트 입력창.
3. Main Content: * PC: 3~4열 그리드 형태의 링크 카드.
    * Mobile: 1열 리스트 또는 2열 바둑판 형태의 큰 터치 카드.
4. Floating Action Button (FAB): 우측 하단 핑크색 '+' 버튼 (링크 추가 모달 연결).

6. 상세 기능 명세
* 모드 필터링: isWorkMode 상태값에 따라 category === 'work'인 링크들만 필터링하여 렌더링.
* 링크 추가 로직: * URL 유효성 검사 (https:// 포함 여부 확인).
    * 입력 완료 시 LocalStorage의 배열에 객체 추가 후 상태 업데이트.
* 모바일 편의성: 모바일 환경에서 링크 추가 시 하단에서 슥 올라오는 Drawer 컴포넌트 적용.
* 즉각적 피드백: 검색창 입력 시 별도의 '검색' 버튼 없이 filter() 함수로 화면 즉시 갱신.

7. 디자인 가이드 (White & Pink + Indigo)
* 공통: 깨끗한 White 배경에 미니멀한 그림자(Shadow) 효과 적용.
* 모드별 테마:
    * Work 모드: Indigo-600 포인트 컬러 사용, 신뢰감 있고 차분한 분위기.
    * Life 모드: Pink-500 포인트 컬러 사용, 화사하고 개인적인 분위기.
* Typography: 숫자와 영문, 한글 가독성이 뛰어난 Pretendard 폰트 적용.
* Touch Target: 모바일 버튼 크기는 최소 60px 이상으로 설정하여 엄지손가락 터치에 최적화.

8. 제약사항
1. 기기 간 동기화 불가: 서버가 없는 로컬 서비스이므로 PC에서 등록한 링크가 폰에 자동으로 나타나지 않음 (JSON 백업으로 보완 가능).
2. 데이터 소멸 주의: 브라우저 '쿠키 및 캐시 삭제' 시 데이터가 초기화될 수 있음을 안내.
3. 파비콘 제한: 일부 사이트의 보안 정책(CORS)으로 인해 파비콘 로드가 실패할 수 있음.
